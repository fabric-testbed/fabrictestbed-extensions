# Explain Module

Explain how a specific part of the FABlib codebase works.

## Arguments

$ARGUMENTS - The module, class, method, or concept to explain.

## Instructions

1. Find the relevant code. Search in:
   - `fabrictestbed_extensions/fablib/` for core classes
   - `fabrictestbed_extensions/cli/` for CLI commands
   - `fabrictestbed_extensions/utils/` for utilities
   - `fabrictestbed_extensions/editors/` for visualization
   - `tests/` for test patterns

2. Read the source code thoroughly.

3. Explain:
   - **What it does** — purpose and responsibility
   - **How it works** — key logic, algorithms, patterns
   - **How it fits** — relationships to other classes/modules
   - **How to use it** — example code snippet
   - **Gotchas** — edge cases, limitations, common mistakes

Keep explanations practical and code-oriented. Reference specific line numbers.
